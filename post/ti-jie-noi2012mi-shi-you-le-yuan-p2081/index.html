题解 [NOI2012]迷失游乐园 洛谷P2081

思路

过于冗长，神犇请自行跳过。

Part I

题意：给定带权的树或基环树，从随机一点出发走每个点至多经过一次的路径，问期望路径长度。

sonu\mathrm{son}_usonu​ 表示 uuu 的子结点数量；fau\mathrm{fa}_ufau​ 表示 uuu 的父结点数量，取值为 111 或 222（因为有环）。

fuf_ufu​ 表示从 uuu 出发，第一步向下走的期望路径长度，「下」指子结点方向即叶子方向；

gug_ugu​ 表示从 uuu 出发，第一步向上走的期望路径长度，「上」指父结点方向。

注意：只规定了第一步向上走，第二步允许去 uuu 的父亲 kkk 的其他子结点。

那么从 uuu 出发的期望路径长度为：

hu=fu⋅sonu+gu⋅fausonu+fau\large h_u=\dfrac{f_u\cdot \mathrm{son}_u+g_u\cdot \mathrm{fa}_u}{\mathrm{son}_u+\mathrm{fa}_u} hu​=sonu​+fau​fu​⋅sonu​+gu​⋅fau​​

Part II

先考虑 50pts 的普通树做法。

fuf_ufu​ 显然只不依赖任何的 ggg 值：

fu=1sonu∑v is son of u(fv+wu,v)\large f_u=\dfrac{1}{\mathrm{son}_u}\sum_{v\text{ is son of }u}(f_v+w_{u,v}) fu​=sonu​1​v is son of u∑​(fv​+wu,v​)

vvv 是 uuu 的子结点，wu,vw_{u,v}wu,v​ 表示边权。特别的，叶子结点 fff 值为 000。

对于普通树，uuu 至多有一个父结点 kkk，而 gug_ugu​ 是要依赖 gk,fkg_k,f_kgk​,fk​ 的：

gu=wu,k+gk⋅fak+fk⋅sonk−fu−wu,kfak+sonk−1\large g_u=w_{u,k}+\dfrac{g_k\cdot \mathrm{fa}_k+f_k\cdot \mathrm{son}_k-f_u-w_{u,k}}{\mathrm{fa}_k+\mathrm{son}_k-1} gu​=wu,k​+fak​+sonk​−1gk​⋅fak​+fk​⋅sonk​−fu​−wu,k​​

解释如下：

不论如何，第一步贡献 wu,kw_{u,k}wu,k​。

粗略地看，继续向上走贡献 gk⋅fakg_k\cdot \mathrm{fa}_kgk​⋅fak​，转而向下走贡献 fk⋅sonkf_k\cdot \mathrm{son}_kfk​⋅sonk​，考虑到不能走回来，故应减去 fu+wu,kf_u+w_{u,k}fu​+wu,k​。

第二步向上有 fak\mathrm{fa}_kfak​ 种选择，向下有 sonk−1\mathrm{son}_k-1sonk​−1 种选择（不允许回来），故分母为 fak+sonk−1\mathrm{fa}_k+\mathrm{son}_k-1fak​+sonk​−1。

特别的，若 kkk 仅与 uuu 相连即 fak+sonk−1=0\mathrm{fa}_k+\mathrm{son}_k-1=0fak​+sonk​−1=0，则 gu=wu,kg_u=w_{u,k}gu​=wu,k​，没有后一项。

实现时，计算 g,fg,fg,f 值，由于前者依赖后者，而后者不依赖前者，故流程如下：

  1. 从根节点开始，递归每个子结点计算 fff 值；
  2. 递归至 uuu 时，先递归子结点，得到每个 fvf_vfv​，然后计算 fuf_ufu​，返回；
  3. 得到该普通树的 fff 值后，从根节点的每个子结点开始，递归计算 ggg 值；
  4. 递归至 uuu 时，根据父结点 kkk 的信息计算 gug_ugu​，然后递归子结点计算，返回；
  5. 根据 gi,fig_i,f_igi​,fi​ 计算 hih_ihi​，最终结果为 1n∑i=1nhi\displaystyle\dfrac{1}{n}\sum_{i=1}^nh_in1​i=1∑n​hi​。

Part III

再来考虑基环树的情况。

如图所示是一棵并不优美的基环树，箭头指向父结点（111 或 222 个），加粗结点是环上结点，以下简称「黑点」，其余结点简称「白点」。

我们将基环树看做若干普通树根结点相连成环的结果。

注意到无论从哪一点出发，第一步向下，活动范围始终在普通树内部，即：无论黑白，fff 值不变。

若 uuu 是白点，只有唯一父亲 kkk，尽管此时 fak\mathrm{fa}_kfak​ 不一定为 111（白点 uuu 的父亲 kkk 是黑点时 fak=2\mathrm{fa}_k=2fak​=2），但显然考虑推导过程，gug_ugu​ 仍符合上文的公式。

若 uuu 是黑点，则 gug_ugu​ 的计算显得和繁琐。

uuu 第一步去了另一个黑点，第二步可以选择继续在环上走，还可以选择钻入该黑点的子树。

为了方便处理，我们先通过深搜得到：

  • 黑点总数 ttt；
  • dfn[u]，维护 uuu 是搜到的第几个黑点；
  • path[i]，维护第 i(1⩽i⩽t)i(1\leqslant i\leqslant t)i(1⩽i⩽t) 个黑点的实际编号 u(1⩽u⩽n)u(1\leqslant u\leqslant n)u(1⩽u⩽n)；
  • disl[i],disr[i]，类似于链表的意味，维护第 iii 个黑点与左右相邻黑点的距离。

对于黑点 uuu，先规定第一步必须逆时针走，则有：

gu=∑v is black point Pv×(fv⋅sonvsonv+1+w)\large g_u=\sum_{v\text{ is black point }}P_v\times \left(\dfrac{f_{v}\cdot \mathrm{son}_{v}}{\mathrm{son}_{v}+1}+w\right) gu​=v is black point ∑​Pv​×(sonv​+1fv​⋅sonv​​+w)

其中，vvv 是 uuu 逆时针方向上依次经历的黑点，即：path[dfn[u]+1] path[dfn[u]+2] ⋯\cdots⋯ path[t] path[1] path[2] ⋯\cdots⋯ path[dfn[u]-1]。

嗯，这个式子有失数学美感，但严谨的讲，的确是这个样子。不要紧，我们来看看这是什么意思。

根据期望的定义，期望等于执行第 kkk 步的概率与第 kkk 步带来的贡献的乘积的累加和。

先来看概率，PPP 表示走到该点的概率，规定了第一步逆时针，即 PPP 初始为 111。

设 vvv 的下一个黑点是 v′v'v′，需要在 sonv\mathrm{son}_{v}sonv​ 个子结点和下一个黑点之中选择下一步去哪里，那么 PPP 的转移关系为：

Pv′=1sonv+1PvP_{v'}=\dfrac{1}{\mathrm{son}_{v}+1}P_v Pv′​=sonv​+11​Pv​

再来看从顺时针方向的上一个黑点走到 vvv 的贡献，首先有边权 www，即 disl[dfn[v]]。

其次，考虑从此钻入 vvv 的子树产生的贡献，即 fv⋅sonvsonv+1\dfrac{f_{v}\cdot \mathrm{son}_{v}}{\mathrm{son}_{v}+1}sonv​+1fv​⋅sonv​​。

特别的，如果 vvv 逆时针方向的下一个黑点就是 uuu 的话，只能选择钻入子树而不能继续走重复的，那么分母就应该为 sonv\mathrm{son}_{v}sonv​ 即钻入该子树的贡献直接就是 fvf_{v}fv​。

刚刚规定了第一步必须逆时针走，现在再规定第一步必须顺时针求一遍，最终给 ggg 值除以 222 即可。

当然，也可以在一开始就令 P=12P=\dfrac{1}{2}P=21​，代码中也是这样实现的。

这很好理解，由于路径不重复，所以不可能顺逆时针交替，顺逆各求一遍一定包含了所有情况，只不过每个概率都放大了 222 倍，现在除以 222 即可。

理清逻辑关系，所有 fff 值只依赖其子结点的 fff 值，黑点的 ggg 值依赖 fff 值，白点的 ggg 值依赖黑点的 ggg 值和 fff 值，故流程如下：

  1. 从每个黑点即每个普通树的根节点开始，递归每个子结点计算 fff 值，方法同普通树；
  2. 计算每个黑点的 ggg 值，顺逆时针各求一遍，然后除以 222，如上文所示；
  3. 从每个黑点的每个子结点开始，递归计算白点的 ggg 值，方法同普通树；
  4. 根据 gi,fig_i,f_igi​,fi​ 计算 hih_ihi​，最终结果为 1n∑i=1nhi\displaystyle\dfrac{1}{n}\sum_{i=1}^nh_in1​i=1∑n​hi​。

代码

带空格有注释，神犇请自行跳过。

#include <cstdio>
const int maxn = 100005;
int n, m, fa[maxn], son[maxn];
double g[maxn], f[maxn], ans;
struct Edge {
	int to, next, w;
	Edge() {}
	Edge(int to, int next, int w): to(to), next(next), w(w) {}
} e[maxn<<1];
int head[maxn], cnt;
void add(int u, int v, int w) {
	e[++cnt] = Edge(v, head[u], w);
	head[u] = cnt;
}
#define v e[i].to
int pos;
bool vis[maxn], flag;//是否在环上
void dfs1(int u, int k) {
	vis[u] = true;
	for(int i = head[u]; i; i = e[i].next)
		if(v != k) {
			if(vis[v]) { pos = v; return; }
			dfs1(v, u);
			if(!flag and pos) { if(pos == u) flag = true; return; }
			if(flag) break;
		}
	vis[u] = false;
}//将所有环上结点的 vis 标记为 true
int t, disl[22], disr[22], dfn[maxn], path[22];
void dfs2(int u, int k) {
	dfn[u] = ++t, path[t] = u;
	fa[u] = 2;
	for(int i = head[u]; i; i = e[i].next)
		if(v != k and vis[v]) {
			if(!dfn[v]) dfs2(v, u);
			disr[dfn[u]] = disl[dfn[v]] = e[i].w;
			break;
		}
}//处理环上信息
void dp_f(int u, int k) {
	for(int i = head[u]; i; i = e[i].next)
		if(v != k and !vis[v]) {
			fa[v] = 1;
			dp_f(v, u);
			son[u]++;
			f[u] += f[v] + e[i].w;
		}
	if(son[u]) f[u] /= son[u];
}
void dp_g(int u, int k, int w) {
	g[u] = w;
	if(fa[k] + son[k] - 1)
		g[u] += (g[k]*fa[k]+f[k]*son[k]-f[u]-w) / (fa[k]+son[k]-1);
	for(int i = head[u]; i; i = e[i].next)
		if(v != k) dp_g(v, u, e[i].w);
}
void work1() {
	dp_f(1, 0);
	for(int i = head[1]; i; i = e[i].next)
		dp_g(v, 1, e[i].w);
}
double P;
#define nxt(x) (x==t?1:x+1)
#define pre(x) (x==1?t:x-1)
void work2() {
	dfs1(1, 0);
	dfs2(pos, 0);
	for(int i = 1; i <= t; ++i)
		dp_f(path[i], 0);
	for(int i = 1, x; i <= t; ++i) {
		x = path[i];
		P = 0.5;
		for(int j = nxt(i), y; j != i; j = nxt(j)) {
			y = path[j];
			if(nxt(j) == i) g[x] += P * (disl[j]+f[y]);
			else g[x] += P * (f[y]*son[y]/(son[y]+1)+disl[j]);
			P /= (son[y]+1);
		}
		P = 0.5;
		for(int j = pre(i), y; j != i; j = pre(j)) {
			y = path[j];
			if(pre(j) == i) g[x] += P * (disr[j]+f[y]);
			else g[x] += P * (f[y]*son[y]/(son[y]+1)+disr[j]);
			P /= (son[y]+1);
		}
		for(int j = head[x]; j; j = e[j].next)
			if(!vis[e[j].to]) dp_g(e[j].to, x, e[j].w);
	}
}
#undef v
int main() {
	scanf("%d %d", &n, &m);
	for(int i = 1, u, v, w; i <= m; ++i)
		scanf("%d %d %d", &u, &v, &w), add(u, v, w), add(v, u, w);
	if(n != m) work1();//普通树
	else work2();//基环树
	for(int i = 1; i <= n; ++i)
		ans += (f[i]*son[i]+g[i]*fa[i]) / (son[i]+fa[i]);
	printf("%.5lf\n", ans / n);
}
