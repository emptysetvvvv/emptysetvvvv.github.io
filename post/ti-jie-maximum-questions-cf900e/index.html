思路

Part I

不考虑实现，胡乱想一个转移方程。

我们用 fif_ifi​ 表示考虑了前 iii 位，最多匹配了多少个串 TTT，gig_igi​ 表示匹配数最大时，至少要使用多少个 ?。

那么对于所有情况，首先有：

fi=fi−1,gi=gi−1f_i=f_{i-1},g_i=g_{i-1} fi​=fi−1​,gi​=gi−1​

边界条件：

f0=0,g0=0f_0=0,g_0=0 f0​=0,g0​=0

将 ? 出现次数的前缀和记为 pip_ipi​

若从 iii 往前长度 mmm 一段可以匹配，则进行更新：

{fi=fi−m+1,gi=gi−m+(pi−pi−m),fi<fi−m+1gi=min⁡(gi−m+(pi−pi−m),gi),fi=fi−m+1do nothing,fi>fi−m+1\begin{cases}f_i=f_{i-m}+1,g_i=g_{i-m}+(p_i-p_{i-m}), &f_{i}<f_{i-m}+1\\g_i=\min\left(g_{i-m}+(p_i-p_{i-m}),g_i\right),& f_i=f_{i-m}+1\\\mathrm{do\ nothing},& f_i>f_{i-m}+1\end{cases} ⎩⎪⎨⎪⎧​fi​=fi−m​+1,gi​=gi−m​+(pi​−pi−m​),gi​=min(gi−m​+(pi​−pi−m​),gi​),do nothing,​fi​<fi−m​+1fi​=fi−m​+1fi​>fi−m​+1​

如果我们每次 O(m)O(m)O(m) 暴力检查可否匹配，总复杂度就会变化为 O(nm)O(nm)O(nm)。

PartII

快速检查能否匹配。

这正是此题的精髓所在，考虑这样一个问题，如果 O(m)O(m)O(m) 匹配，题目完全没有必要保证 TTT 仅由 a b 组成，完全可以直接给出，更没必要规定为 a b 交替出现。

敲黑板，我们用 h1,ih_{1,i}h1,i​ 表示从 iii 向前最长有多长一段是 a b 交替出现，且以 a 结尾的串。对应的，用 h0,ih_{0,i}h0,i​ 表示 b 结尾的。

则有：

h0,i=h1,i−1+1(si=a)h_{0,i}=h_{1,i-1}+1(s_i\not =\texttt{a}) h0,i​=h1,i−1​+1(si​​=a)

h1,i=h0,i−1+1(si=b)h_{1,i}=h_{0,i-1}+1(s_i\not =\texttt{b}) h1,i​=h0,i−1​+1(si​​=b)

那么判定能否匹配，就是说是否有 hmand⁡2,i⩾mh_{m\operatorname{and}2,i}\geqslant mhmand2,i​⩾m，复杂度 O(1)O(1)O(1)，总复杂度 O(n)O(n)O(n)。

代码

#include <cstdio>
#include <iostream>
using namespace std;
const int maxn = 100005;
int n, m, p[maxn], g[maxn], f[maxn], h[2][maxn];
char s[maxn];
int main() {
	scanf("%d %s %d", &n, s+1, &m);
	for(int i = 1; i <= n; ++i) {
		p[i] = p[i-1] + (s[i] == '?');
		if(s[i] != 'a') h[0][i] = h[1][i-1] + 1;
		if(s[i] != 'b') h[1][i] = h[0][i-1] + 1;
		f[i] = f[i-1], g[i] = g[i-1];
		if(h[m&1][i] >= m) {
			if(f[i] < f[i-m]+1) f[i] = f[i-m]+1, g[i] = g[i-m]+p[i]-p[i-m];
			else if(f[i] == f[i-m]+1) g[i] = min(g[i], g[i-m]+p[i]-p[i-m]);
		}
	}
	printf("%d\n", g[n]);
}
